Key Changes in v3.1:
--------------------
1. Simplified Configuration:
   - OLD: cluster-name.management-cluster.provisioner
   - NEW: cluster-name (one per line)
   - 60% less configuration effort

2. Auto-Discovery:
   - Automatically discovers management cluster from TMC
   - Automatically discovers provisioner from TMC
   - Caches metadata for performance

3. Auto-Context Creation:
   - Automatically creates TMC contexts based on naming patterns
   - *-prod-[1-4] → Production TMC context
   - *-uat-[1-4] or *-system-[1-4] → Non-production TMC context
   - Reuses existing contexts for efficiency

4. Unified Execution:
   - No more --multi flag
   - Always reads from clusters.conf
   - Single execution path

5. Code Reduction:
   - 33% less code (from ~1070 to ~720 lines)
   - Improved maintainability
   - Better error handling

Usage Examples:
---------------
# PRE-change health check
./k8s-health-check-pre.sh ./clusters.conf

# POST-change health check with comparison
./k8s-health-check-post.sh ./clusters.conf ./health-check-results/pre-20250122_143000

# With TMC credentials in environment
TMC_SELF_MANAGED_USERNAME=user TMC_SELF_MANAGED_PASSWORD=pass ./k8s-health-check-pre.sh ./clusters.conf

# With debug output
DEBUG=on ./k8s-health-check-pre.sh ./clusters.conf

Prerequisites:
--------------
- Tanzu CLI installed and in PATH
- kubectl installed and configured
- TMC self-managed instance access
- Cluster naming patterns: *-prod-[1-4], *-uat-[1-4], *-system-[1-4]
- jq (optional but recommended for better performance)

Configuration Required:
-----------------------
1. Edit lib/tmc-context.sh:
   - Set NON_PROD_DNS to your non-prod TMC FQDN
   - Set PROD_DNS to your prod TMC FQDN

2. Create clusters.conf:
   - Add cluster names (one per line)
   - Follow naming patterns for auto-context detection

Environment Variables (optional):
